# تعليمات النشر المؤكدة — KAG Operational Project Management UI

هذه الحزمة هي النسخة المؤكدة للواجهة السينمائية التنفيذية الجديدة.

## مهم جدًا
لا ترفع ملف ZIP نفسه داخل GitHub فقط. يجب فك الضغط ورفع محتويات الحزمة إلى جذر المستودع.

يجب أن تظهر هذه الملفات مباشرة في Root Repository:

- server.js
- package.json
- render.yaml
- public/index.html
- public/style.css
- public/script.js

إذا كان GitHub يحتوي ملف ZIP فقط أو يحتوي الملفات داخل مجلد داخلي، فلن يفتح Render الواجهة الجديدة.

## الصفحة الرئيسية الصحيحة
المسار الصحيح بعد النشر:

/

أو:

/index.html

والخادم يخدم تلقائيًا الملف:

public/index.html

## أوامر Render
Build Command:

npm install && pip install -r requirements.txt

Start Command:

node server.js

Health Check Path:

/api/health

## التحقق بعد النشر
افتح:

/api/health

يجب أن يرجع JSON فيه ok:true.

ثم افتح:

/

إذا ظهر داشبورد قديم، فالسبب غالبًا أحد الآتي:

1. لم يتم فك ضغط الحزمة داخل GitHub.
2. الملفات الجديدة موجودة داخل مجلد فرعي وليست في الجذر.
3. Render مربوط بفرع مختلف.
4. Render لم يعمل Clear build cache.
5. خدمة Render تستخدم Root Directory مختلف.
6. المتصفح يعرض Cache قديم؛ جرّب Hard Refresh.

## إعداد مؤقت للمشاهدة بدون تسجيل دخول
للمعاينة فقط يمكن وضع:

REQUIRE_LOGIN=false

ثم بعد الاعتماد يرجع:

REQUIRE_LOGIN=true
