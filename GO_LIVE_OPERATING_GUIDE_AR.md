# دليل تشغيل مركز الإطلاق التشغيلي

## 1. قبل الرفع على GitHub
- فك ضغط الحزمة.
- ارفع محتويات الحزمة نفسها، وليس ملف ZIP.
- تأكد أن الملفات التالية موجودة في الجذر:
  - `server.js`
  - `package.json`
  - `render.yaml`
  - `public/index.html`
  - `public/style.css`
  - `public/script.js`

## 2. إعداد Render
Build Command:
```bash
npm install && pip install -r requirements.txt
```

Start Command:
```bash
node server.js
```

Health Check Path:
```txt
/api/health
```

## 3. بعد النشر
افتح المنصة ثم ادخل كأدمن، ثم راجع الصفحات التالية بالترتيب:
1. حالة النظام.
2. جاهزية النشر والتشغيل.
3. مركز الإطلاق التشغيلي.
4. جودة البيانات.
5. مركز الإشعارات.
6. مركز المهام والمتابعة.

## 4. بوابات القبول
لا تعرض المنصة رسميًا قبل تحقق الآتي:
- درجة جاهزية النشر فوق 90%.
- درجة جودة البيانات فوق 85%.
- اختبار إرسال إشعار داخلي ناجح.
- تحويل إشعار إلى مهمة ناجح.
- إغلاق مهمة بدليل ناجح.
- تنزيل نسخة احتياطية JSON ناجح.
- تصدير CSV للمهام والاعتمادات ناجح.
- تجربة الجوال ناجحة.

## 5. خطة تشغيل أول أسبوع
اليوم 1: ربط البيانات الحقيقية.
اليوم 2: إرسال إشعارات فعلية.
اليوم 3: إغلاق مهام بأدلة.
اليوم 4: اختبار التصعيد والاعتمادات.
اليوم 5: توليد تقرير تشغيلي.
اليوم 6: اختبار الجوال و PWA.
اليوم 7: اعتماد النسخة المستقرة.
