# سجل تعديل آلية التقارير — PDF / PowerPoint

تم تعديل آلية التصدير بحيث لا يتم تسليم المستخدم النهائي ملفات Excel أو CSV من مركز التقارير.

## ما الذي تغيّر؟
- أصبح التقرير الافتراضي يصدر مباشرة بصيغة PDF جاهزة للطباعة والعرض.
- تم إضافة خيار PowerPoint قابل للتعديل لكل تقرير.
- تم إلغاء توليد CSV من أزرار مركز التقارير في الواجهة.
- تم ربط `/api/report` بمعامل `format`:
  - `pdf` لإنتاج ملف PDF.
  - `pptx` لإنتاج ملف PowerPoint.
- يتم تمرير جميع بيانات النظام إلى محرك التقارير، وتشمل:
  - المسارات والمهام.
  - الإشعارات.
  - المهام الناتجة من الإشعارات.
  - الاعتمادات.
  - الأدلة الميدانية.
  - الاجتماعات.
  - المناطق.
  - جودة البيانات.
  - الملخص التشغيلي.
- عند طلب PowerPoint يستخدم النظام القوالب المعتمدة أو القوالب الاحتياطية داخل `report_engine`.
- عند طلب PDF يحاول النظام التحويل من PowerPoint، وفي حال كانت البيئة لا تدعم التحويل أو كان القالب كبيرًا يستخدم مولد PDF مباشر بنفس الهوية التنفيذية.

## نقطة مهمة
روابط CSV الخاصة بالتصدير الإداري مثل `/api/export/actions.csv` ما زالت موجودة لاستخدام الأدمن والنسخ الاحتياطي فقط، وليست جزءًا من مركز التقارير النهائي للمستخدم.

## طريقة الاستخدام
من الواجهة:
- اضغط PDF للحصول على تقرير جاهز للطباعة.
- اضغط PowerPoint للحصول على ملف قابل للتعديل.

من API:
```bash
curl -X POST https://YOUR-APP.onrender.com/api/report \
  -H "Content-Type: application/json" \
  -d '{"type":"executive","format":"pdf"}' \
  -o report.pdf

curl -X POST https://YOUR-APP.onrender.com/api/report \
  -H "Content-Type: application/json" \
  -d '{"type":"executive","format":"pptx"}' \
  -o report.pptx
```
