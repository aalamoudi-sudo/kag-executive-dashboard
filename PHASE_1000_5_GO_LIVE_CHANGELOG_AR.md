# سجل تحديث المرحلة 1000.5 — مركز الإطلاق التشغيلي

## الهدف
رفع المنصة من جاهزية النشر إلى جاهزية التشغيل الفعلي، بحيث يمتلك الفريق التقني ومدير المشروع بوابات قبول واضحة قبل العرض أو التشغيل الرسمي.

## التعديلات الجديدة
- إضافة صفحة جديدة داخل القائمة: **مركز الإطلاق التشغيلي**.
- إضافة API جديد: `GET /api/go-live-control`.
- إضافة درجة إطلاق مركبة تجمع:
  - جاهزية النشر.
  - جودة البيانات.
  - قابلية تشغيل دورة العمل.
- إضافة بوابات قبول تشغيلية:
  - رفع الملفات مفكوكة على GitHub.
  - تفعيل تسجيل الدخول والصلاحيات.
  - جاهزية مصدر البيانات.
  - جاهزية قناة إشعار خارجية.
  - قابلية اختبار دورة التشغيل.
  - جودة البيانات.
  - النسخ والتصدير.
- إضافة خطة تشغيل أول أسبوع.
- إضافة قائمة تسليم للفريق.
- إضافة مخاطر قبول قبل العرض وطريقة التعامل معها.
- إصلاح دوال الواجهة الخاصة بصفحة جاهزية النشر واختبار التكاملات.

## ملاحظات مهمة
- صفحة مركز الإطلاق التشغيلي تتطلب صلاحية أدمن.
- في وضع التطوير يمكن تشغيل `REQUIRE_LOGIN=false` لاختبار الصفحة محليًا.
- واتساب يحتاج مزود WhatsApp Business API أو Webhook وسيط، والمنصة جاهزة للربط وليست مزود إرسال مباشر.
